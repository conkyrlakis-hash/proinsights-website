# ProInsights — Marketing Website

The public homepage at `https://proinsights.com.cy` — pure HTML/CSS, no framework, deploys to Cloudflare Pages in 5 minutes.

## File map

```
proinsights-website/
├── index.html              # Homepage (hero, features, pricing, FAQ, CTA)
├── styles.css              # All styling — brand-locked navy + gold
├── sample-hotel.html       # Live demo report — Amarande Hotel (Expert tier)
├── sample-restaurant.html  # Live demo report — LA VIE Düsseldorf (Pro tier)
├── README.md               # This file
└── (optional) privacy.html / terms.html / gdpr.html — to add when launching
```

## Local preview

Just open `index.html` in any browser. There's no build step.

```bash
# Or serve with Python's built-in server if you want clean URLs
cd proinsights-website
python -m http.server 8000
# → http://localhost:8000
```

## Deploy to Cloudflare Pages

### 1. Push to a new GitHub repo

```bash
cd proinsights-website
git init
git add .
git commit -m "Initial commit: ProInsights marketing site"
git branch -M main
gh repo create conkyrlakis-hash/proinsights-website --public --source=. --push
```

(Or if `gh` isn't installed, create the repo manually at github.com/new and push.)

### 2. Connect Cloudflare Pages

1. Cloudflare dashboard → **Workers & Pages** → **Create application** → **Pages** → **Connect to Git**
2. Authorise GitHub → select `proinsights-website`
3. Build settings:
   - **Framework preset:** None
   - **Build command:** *(leave blank — pure static)*
   - **Build output directory:** `/`
4. **Save and Deploy**

First deploy is ~30 seconds. You'll get a URL like `proinsights-website.pages.dev` — confirm it loads correctly.

### 3. Bind to the apex domain

1. In the Pages project → **Custom domains** → **Set up a custom domain**
2. Enter: `proinsights.com.cy`
3. Cloudflare detects you own the parent domain → click **Activate**
4. Cloudflare auto-creates the CNAME flattening for the apex
5. Add the `www` subdomain too: → **Custom domains** → enter `www.proinsights.com.cy`

Within 1-2 minutes both `https://proinsights.com.cy` and `https://www.proinsights.com.cy` should load the site.

### 4. (Optional) Set up redirects

Create a `_redirects` file (Cloudflare Pages syntax) so common typos / old URLs route cleanly:

```
# /_redirects
/home            /                    301
/sample          /sample-hotel.html   301
/demo            /#contact            301
/pricing.html    /#pricing            301
```

Commit + push, Pages auto-redeploys.

## Editing copy

The two files you edit most:

- **`index.html`** — copy, headings, FAQ, pricing tiers
- **`styles.css`** — colors, type, spacing

Brand colors in CSS variables at the top of `styles.css`:

```css
:root {
  --navy: #0a2540;
  --gold: #C89B3C;
  --ink: #1a1a2e;
  ...
}
```

Change in one place; updates everywhere.

## Sample reports

The two `sample-*.html` files are generated by `proinsights-reports/build_demo_report.py`. Regenerate them whenever you want fresher data:

```bash
cd ../proinsights-reports
python build_demo_report.py \
    --vertical hotel \
    --venue "Amarande Hotel" \
    --competitors "Anassa,Olympic Lagoon,Adams Beach,Grecian Bay" \
    --tier expert \
    --output ../proinsights-website/sample-hotel.html

python build_demo_report.py \
    --vertical restaurant \
    --venue "LA VIE by Thomas Bühner" \
    --competitors "Setzkasten,Im Schiffchen,Nagaya,Agata's,Jae,Yoshi by Nagaya,1876 Daniel Dal-Ben,Zwanzig23" \
    --tier pro \
    --output ../proinsights-website/sample-restaurant.html
```

Then commit + push — Cloudflare auto-redeploys.

## Demo request email

Hero "Book a demo" + CTA banner button use `mailto:hello@proinsights.com.cy?subject=Demo request&body=...`. This is fine for v1.

When volume picks up, replace with a proper form:
- **Option A:** Cloudflare Pages Functions + Resend / SendGrid email forward (free, native)
- **Option B:** Tally / Typeform embed (easier, no code)
- **Option C:** Calendly inline embed (best — books a real slot in your calendar)

Pick Option C when the first 3 inbound demos land — saves you the email tag.
